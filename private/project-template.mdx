# Guidelines for Project Metadata:

## Title
- Keep it concise and descriptive
- Use title case (capitalize important words)
- Avoid generic terms like "Project" or "App"

## Slug
- Use kebab-case (lowercase with hyphens)
- Should match the filename (without .mdx extension)
- Keep it short and memorable

## Description
- Write 1-2 compelling sentences
- Highlight key features and technologies
- Focus on the value proposition
- Use action words and specific details
- Avoid vague terms like "amazing" or "great"

## Cover Images
- Add 1-5 high-quality screenshots
- Use descriptive filenames (cover-01.png, cover-02.png, etc.)
- Place images in /public/projects/project-slug/
- Recommended size: 1200x800px or similar aspect ratio

## Icon
- Use a square icon (preferably 512x512px)
- Place in /public/projects/project-slug/icon.png
- Should represent the project visually
- Use iconSize: "fit" for logos, leave empty for square icons

## Category
- Choose from: Web Development, Mobile App, IoT, AI/ML, Education Technology, E-commerce, SaaS, Open Source, Design, Game Development, etc.
- Be consistent with existing categories

## Dates
- Use YYYY-MM-DD format
- startDate: When you started working on the project
- endDate: When project was completed (null for ongoing)

## Tech Stack
- List 5-8 most important technologies
- Order by importance/usage
- Use official names (e.g., "Next.js 14", "TypeScript", "PostgreSQL")
- Include frameworks, languages, databases, tools, services

## Role
- Be specific about your responsibilities
- Examples: "Full-stack Developer", "Frontend Developer & UI Designer", "Lead Developer", "Solo Developer"

## Status
- Planning: Project is in planning/design phase
- In Development: Actively being built
- In Production: Live and being used
- Completed: Finished and no longer maintained
- Archived: No longer active

## Links
- website: Main project URL
- repository: GitHub/GitLab repository (empty string if private)
- demo: Demo or preview URL (empty string if none)

## Selected
- true: Will appear in featured projects on homepage
- false: Will only appear in projects page
- Choose your best 2-3 projects for featured section
